# 素材指定经销商分发设计

## 背景

Apple Lead 在活动准备阶段需要将游戏公司制作好的 LED 等素材包分发给参与活动的经销商。当前系统上传后的素材对所有经销商可见，只能通过文件名中的经销商名称做人工区分，容易误下载、误使用。

## 已确认方案

采用方案 A：一次上传的一批文件共用同一组分发经销商。

- 上传弹窗新增必填字段：分发经销商。
- 分发经销商支持单选和多选。
- 可选范围仅为参与当前活动的经销商。
- 一次上传多个文件时，本次上传的所有文件共用同一组分发对象。
- 素材记录仍按文件粒度保存 `dealerIds`，便于后续扩展到“每个文件单独指定经销商”。
- Apple Lead 的素材表格展示分发对象。
- 经销商素材下载页按当前经销商身份过滤，只展示分发给自己的素材。

## UI 设计

参考 `D:\Codex Project\小红书看板` 的 React + TypeScript + Vite + DevExpress 技术栈和视觉规范：

- 浅灰页面背景。
- 白色卡片和轻边框。
- 8px 圆角。
- Apple 风格字体栈。
- `#0071e3` 作为主操作色。
- 使用 DevExpress React 组件实现表格、弹窗、多选、上传与按钮。

## 关键交互

1. Apple Lead 点击“上传素材”。
2. 弹窗打开后，在“分发经销商”中选择一个或多个参与活动经销商。
3. 拖拽或点击上传一个或多个文件。
4. 点击确认后，系统为每个文件创建一条素材记录，并写入同一组 `dealerIds`。
5. Apple Lead 表格展示所有素材及分发对象。
6. 经销商进入素材下载页时，按当前经销商 ID 过滤可见素材。

## 后续扩展点

- 支持每个文件单独设置分发经销商。
- 支持“全部参与经销商”作为快捷选择。
- 支持上传后再次调整分发对象。
- 支持分发状态、下载状态与经销商确认回执。
